I start by installing **QSSTV** using the command:
```
sudo apt-get install qsstv
```

Next, I create a virtual audio output by running the following command to load the **module-null-sink** in **PulseAudio**:
```
pactl load-module module-null-sink sink_name=virtual-cable
```

Now, I open **pavucontrol** by typing `pavucontrol` in the terminal. When the GUI opens up, I go to the **Output Devices** tab and verify that I see the **"Null Output"** device listed.

I launch **QSSTV** by simply typing:
```
qsstv
```

Once the QSSTV program starts up, I head over to **Options** → **Configuration** → **Sound** and make sure I select **PulseAudio** as the audio interface.

Back in the **pavucontrol** GUI, I navigate to the **Recording** tab. Here, I make sure that **QSSTV** is capturing audio from the **Null Output** (or **Monitor of virtual-cable**, depending on what I see).

The hint that asks "What is the CMU mascot?" pointed to the answer being **"Scotty the Scottie Dog"**. So, I set the **QSSTV** mode to **"Scottie 1"**, and I also check the **Auto Slant** option to let it automatically correct any slanting.

Finally, I run **paplay** to play the `message.wav` file through the virtual cable and create the image:
```
paplay -d virtual-cable message.wav
```
